# 安装与配置

## 安装

### 使用 pnpm（推荐）

```bash
pnpm add -D @ai-partner-x/eslint-plugin-aiko-boot @typescript-eslint/parser
```

### 使用 npm

```bash
npm install -D @ai-partner-x/eslint-plugin-aiko-boot @typescript-eslint/parser
```

### 使用 yarn

```bash
yarn add -D @ai-partner-x/eslint-plugin-aiko-boot @typescript-eslint/parser
```

## 配置方式

### 方式一：使用预设配置（推荐）

创建 `.eslintrc.json` 文件：

```json
{
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2022,
    "sourceType": "module"
  },
  "plugins": ["@ai-partner-x/aiko-boot"],
  "extends": ["plugin:@ai-partner-x/aiko-boot/recommended"]
}
```

### 方式二：使用 java-compat 配置

适用于需要转译到 Java 的项目：

```json
{
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2022,
    "sourceType": "module"
  },
  "plugins": ["@ai-partner-x/aiko-boot"],
  "extends": ["plugin:@ai-partner-x/aiko-boot/java-compat"]
}
```

### 方式三：自定义规则配置

```json
{
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2022,
    "sourceType": "module"
  },
  "plugins": ["@ai-partner-x/aiko-boot"],
  "rules": {
    "@ai-partner-x/aiko-boot/no-arrow-methods": "error",
    "@ai-partner-x/aiko-boot/no-destructuring-in-methods": "error",
    "@ai-partner-x/aiko-boot/no-object-spread": "warn",
    "@ai-partner-x/aiko-boot/static-route-paths": "error",
    "@ai-partner-x/aiko-boot/require-rest-controller": "error"
  }
}
```

## 与其他配置结合

### 结合 TypeScript ESLint

```json
{
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "ecmaVersion": 2022,
    "sourceType": "module",
    "project": "./tsconfig.json"
  },
  "plugins": ["@typescript-eslint", "@ai-partner-x/aiko-boot"],
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended",
    "plugin:@ai-partner-x/aiko-boot/java-compat"
  ]
}
```

### 结合 Prettier

```json
{
  "parser": "@typescript-eslint/parser",
  "plugins": ["@ai-partner-x/aiko-boot"],
  "extends": [
    "plugin:@ai-partner-x/aiko-boot/recommended",
    "prettier"
  ]
}
```

## 针对特定目录配置

如果只想对特定目录（如 `src/`）启用规则：

```json
{
  "parser": "@typescript-eslint/parser",
  "overrides": [
    {
      "files": ["src/**/*.ts"],
      "plugins": ["@ai-partner-x/aiko-boot"],
      "extends": ["plugin:@ai-partner-x/aiko-boot/java-compat"]
    }
  ]
}
```

## 运行检查

### 命令行

```bash
# 检查所有文件
npx eslint src/

# 检查并自动修复
npx eslint src/ --fix

# 仅检查特定文件
npx eslint src/controller/*.ts
```

### package.json 脚本

```json
{
  "scripts": {
    "lint": "eslint src/",
    "lint:fix": "eslint src/ --fix"
  }
}
```

## 禁用规则

### 禁用单行

```typescript
// eslint-disable-next-line @ai-partner-x/aiko-boot/no-destructuring-in-methods
const { id, name } = user;
```

### 禁用代码块

```typescript
/* eslint-disable @ai-partner-x/aiko-boot/no-object-spread */
const merged = { ...obj1, ...obj2 };
/* eslint-enable @ai-partner-x/aiko-boot/no-object-spread */
```

### 禁用整个文件

在文件开头添加：

```typescript
/* eslint-disable @ai-partner-x/aiko-boot/no-arrow-methods */
```

## 常见问题

### Q: 为什么需要 @typescript-eslint/parser？

A: 本插件的规则需要解析 TypeScript 语法，`@typescript-eslint/parser` 提供了这个能力。

### Q: recommended 和 java-compat 有什么区别？

A: `recommended` 包含基础的 Java 兼容规则，而 `java-compat` 包含所有转译相关规则，更加严格。如果你的项目需要转译到 Java，建议使用 `java-compat`。

### Q: 如何只对部分文件启用规则？

A: 使用 `overrides` 配置，可以针对特定文件模式应用不同的规则集。

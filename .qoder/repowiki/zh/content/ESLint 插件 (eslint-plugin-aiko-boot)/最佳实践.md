# 最佳实践

本文档介绍使用 `@ai-partner-x/eslint-plugin-aiko-boot` 的最佳实践，帮助你编写高质量、可转译的 TypeScript 代码。

## 项目配置建议

### 新项目

对于新项目，建议直接使用 `java-compat` 配置：

```json
{
  "extends": ["plugin:@ai-partner-x/aiko-boot/java-compat"]
}
```

### 现有项目迁移

对于现有项目，建议分阶段迁移：

**阶段 1**：先使用 `recommended` 配置

```json
{
  "extends": ["plugin:@ai-partner-x/aiko-boot/recommended"]
}
```

**阶段 2**：修复所有错误后，升级到 `java-compat`

```json
{
  "extends": ["plugin:@ai-partner-x/aiko-boot/java-compat"]
}
```

## 编码规范

### 类方法定义

```typescript
// ✅ 推荐：标准方法定义
@Service()
class UserService {
  async getUserById(id: number): Promise<User | null> {
    return this.userMapper.selectById(id);
  }
  
  async createUser(dto: CreateUserDto): Promise<User> {
    const user = new User();
    user.username = dto.username;
    user.email = dto.email;
    return this.userMapper.insert(user);
  }
}
```

### 数据访问

```typescript
// ✅ 推荐：显式属性访问
async processUser(dto: UserDto): Promise<void> {
  const username = dto.username;
  const email = dto.email;
  const age = dto.age;
  
  // 处理逻辑
}

// ❌ 避免：解构赋值
async processUser(dto: UserDto): Promise<void> {
  const { username, email, age } = dto;  // 不兼容 Java
}
```

### 条件判断

```typescript
// ✅ 推荐：显式 null 检查
async getUsername(user: User | null): Promise<string | null> {
  if (user !== null) {
    return user.username;
  }
  return null;
}

// ✅ 推荐：三元运算符替代空值合并
async getPage(params: SearchParams): Promise<number> {
  const page = params.page !== undefined ? params.page : 1;
  return page;
}

// ❌ 避免：可选链和空值合并
async getUsername(user: User | null): Promise<string | null> {
  return user?.username ?? null;  // 不兼容 Java
}
```

### 类型定义

```typescript
// ✅ 推荐：使用接口定义类型
interface UserSearchParams {
  username: string;
  email: string;
  page: number;
  pageSize: number;
}

interface UserSearchResult {
  data: User[];
  total: number;
}

async searchUsers(params: UserSearchParams): Promise<UserSearchResult> {
  // ...
}

// ❌ 避免：内联对象类型
async searchUsers(params: { username: string }): Promise<{ data: User[] }> {
  // 不兼容 Java
}
```

### 返回类型

```typescript
// ✅ 推荐：显式返回类型和变量声明
async search(params: SearchParams): Promise<SearchResult> {
  const data = await this.userMapper.selectList();
  const total = await this.userMapper.count();
  
  const result: SearchResult = { data, total };
  return result;
}

// ❌ 避免：内联返回对象
async search(params: SearchParams): Promise<SearchResult> {
  return {  // 直接返回对象字面量可能导致转译问题
    data: await this.userMapper.selectList(),
    total: await this.userMapper.count(),
  };
}
```

## 控制器规范

### 路由定义

```typescript
// ✅ 推荐：静态字符串路径
@RestController({ path: '/api/users' })
class UserController {
  @GetMapping('/:id')
  async getById(@PathVariable('id') id: string): Promise<User | null> {
    return this.userService.getUserById(Number(id));
  }
  
  @PostMapping()
  async create(@RequestBody() dto: CreateUserDto): Promise<User> {
    return this.userService.createUser(dto);
  }
}
```

### DTO 和响应类型

```typescript
// ✅ 推荐：定义明确的 DTO 和响应类型
export interface CreateUserDto {
  username: string;
  email: string;
  age: number;
}

export interface SuccessResponse {
  success: boolean;
}

export interface DeleteResponse {
  success: boolean;
  deleted: number;
}

@RestController({ path: '/api/users' })
class UserController {
  @DeleteMapping('/:id')
  async delete(@PathVariable('id') id: string): Promise<SuccessResponse> {
    const result = await this.userService.deleteUser(Number(id));
    const response: SuccessResponse = { success: result };
    return response;
  }
}
```

## 常见问题解决

### 问题：大量解构赋值需要修改

**解决方案**：使用 VS Code 的批量替换功能，或编写脚本自动转换。

### 问题：联合类型广泛使用

**解决方案**：将联合类型替换为基础类型（如 `string`），并在注释中说明可选值。

### 问题：返回内联对象

**解决方案**：定义接口，使用变量声明后返回。

```typescript
// 修改前
return { data, total };

// 修改后
const result: SearchResult = { data, total };
return result;
```

## 总结

遵循以下原则编写 Java 兼容的 TypeScript 代码：

1. **使用标准方法定义**，避免箭头函数作为类方法
2. **使用显式属性访问**，避免解构赋值
3. **使用三元运算符**，避免可选链和空值合并
4. **定义明确的接口**，避免内联对象类型
5. **使用静态字符串路径**，避免动态路由
6. **显式声明返回类型**，确保类型安全
